### 1. ✅ Finding #1 (Medium): RefreshError messaging

**Implementation:** [load_or_acquire_credentials](ergodix/oauth.py#L416-L443)

When a refresh token refresh fails, the code now surfaces the underlying `RefreshError` message via `output_fn` before clearing tokens and re-acquiring:

```python
except RefreshError as exc:

        f"Re-authenticating. If this persists, check your network "

```

- ✅ Passes

**Impact:** Users now see why they're being re-prompted (revoked token, network issue, bad config) instead of silent restart. Migrate's caller can distinguish and report these paths clearly.

### 2. ✅ Finding #2 (Low): Client-ID consistency check

**Implementation:** [_client_id_matches_config](ergodix/oauth.py#L353-L385)

New helper function validates that the token's `client_id` matches the current `google_oauth_client_id` config before any refresh attempt:

```python
def _client_id_matches_config(creds: Credentials, output_fn: OutputFn) -> bool:
    """Return True if creds.client_id matches the current config's
    google_oauth_client_id, else False."""
    try:
        current_client_id = get_credential("google_oauth_client_id")
    except RuntimeError:
        return True  # Config not set yet — let downstream handle it
    
    if creds.client_id == current_client_id:
        return True
    
    output_fn(f"OAuth client ID changed...")
    return False
```

**Called from:** [load_or_acquire_credentials](ergodix/oauth.py#L420-L423)

```python
if not _client_id_matches_config(creds, output_fn):
    clear_oauth_tokens()
```

**Test coverage:**
- `test_load_or_acquire_clears_stale_tokens_on_client_id_mismatch`: token has "old-ci", config has "new-ci" → cleared with message, fresh acquire runs ✅
- `test_load_or_acquire_skips_client_id_check_when_config_unset`: config unset (RuntimeError) → check skipped, proceeds normally ✅

**Impact:** If a user rotates their OAuth client credentials (moved GCP project, regenerated secrets), stale tokens are cleared immediately with a clear message. Prevents the refresh from burning a refresh-token attempt that's guaranteed to fail.

### 3. ✅ Finding #5 (Low): Parent-dir mode safety net in save_oauth_tokens

**Implementation:** [save_oauth_tokens](ergodix/oauth.py#L150-L166)

Parent directory mode enforcement added on both creation and validation:

```python
parent.mkdir(parents=True, exist_ok=True, mode=0o700)
_check_parent_dir_mode(parent)
```

The `mode=0o700` applies only to newly-created directories (per Path.mkdir semantics). The subsequent `_check_parent_dir_mode()` call validates any pre-existing parent, so:
- Fresh installs: parent created at 0o700, check passes
- Pre-existing 0o700 parent: check passes
- Pre-existing 0o755 parent: check raises with remediation message

**Test coverage:** `test_save_rejects_loose_parent_dir`
- Sets parent to 0o755 (loose)
- Calls save_oauth_tokens()
- Verifies PermissionError is raised with "loose permissions" message
- Confirms no token file is created in unsafe parent
- ✅ Passes

**Impact:** Mirrors the load-side check from ADR 0015 §1. A 0o755 parent (the default umask result) silently undermines the file's 0o600 invariant. Now the failure is loud, with clear remediation.

### 4. ✅ Finding #6 (Low): Broken local_config.py warning

**Implementation:** [_token_file_path](ergodix/oauth.py#L46-L76)

Changed from `contextlib.suppress(Exception)` to explicit try/except that emits a `UserWarning` on import failure:

```python
try:
    module = importlib.util.module_from_spec(spec)
    spec.loader.exec_module(module)
except Exception as exc:
    warnings.warn(
        (
            f"local_config.py at {config_path} could not be loaded "
            f"({type(exc).__name__}: {exc}); falling back to default "
            f"token file path. Any TOKEN_FILE override in local_config.py "
            f"is being ignored until you fix the file."
        ),
        UserWarning,
        stacklevel=2,
    )
```

**Test coverage:**
- `test_token_file_path_warns_when_local_config_is_broken`: broken syntax in local_config.py → warning emitted, fallback used ✅
- `test_token_file_path_no_warning_when_local_config_clean`: valid local_config.py with TOKEN_FILE → no warning, custom path returned ✅

**Impact:** If a user's `local_config.py` has a syntax error or broken import, they now see a warning (visible via `--verbose` or pytest) instead of silently using the wrong token path. The fallback still works so OAuth doesn't crash, but the diagnostic is loud.
