# PR Review — ADR 0015.3

Scope reviewed since [reviews/0015.2.external-review.md](reviews/0015.2.external-review.md): merged PRs #82, #83, #84, #85 on [main](main).

## Findings

1. Medium: New smoke test currently fails on main due version mismatch and is likely too brittle around version source selection and sync assumptions.
File references: [scripts/integration-smoke.sh](scripts/integration-smoke.sh#L111), [scripts/integration-smoke.sh](scripts/integration-smoke.sh#L113), [scripts/integration-smoke.sh](scripts/integration-smoke.sh#L114)

Observed run:
- Command: bash scripts/integration-smoke.sh
- Prompt answer used: n at cantilever consent gate
- Result: FAIL: version mismatch — VERSION file says 1.62.1, ergodix --version says 1.60.0

Additional inspection after failure:
- Source VERSION: 1.62.1
- Deploy VERSION at /tmp/ergodix-smoke-deploy/VERSION: 1.60.0
- Installed dist metadata in deploy venv: 1.62.1

This means the check at [scripts/integration-smoke.sh](scripts/integration-smoke.sh#L111-L115) can fail even when the installed package metadata is current, so the script is currently producing a false-negative signal in at least one realistic local path.

Recommendation:
- Compare against deployed source of truth, not workspace source path:
  - expected from DEPLOY_DIR/VERSION, or
  - expected from python -m importlib.metadata version check in the same venv.
- Keep both checks if desired, but emit separate diagnostics so mismatch source is explicit.

2. Low: Smoke script currently blocks on interactive cantilever consent, so it is not fully non-interactive for CI without additional flags.
File references: [scripts/integration-smoke.sh](scripts/integration-smoke.sh#L92), [scripts/integration-smoke.sh](scripts/integration-smoke.sh#L94)

The script is documented as CI-ready, but bootstrap invokes cantilever consent interactively. In this review run, an input prompt appeared and required explicit user choice.

Recommendation:
- Add a non-interactive mode for smoke use, either by:
  - forwarding a CI/non-interactive arg to bootstrap if supported, or
  - splitting bootstrap-install verification from cantilever apply for smoke mode.
